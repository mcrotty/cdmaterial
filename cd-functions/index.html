We have used functions before, however, we have not explained why we use them.

Functions allow us:

  1. To allow reuse of code! Instead of having to type the code in again, we create a function with a name, move the code into it, and then call it as often as we need it.
  2. To break the code into easier to manage pieces. Rather than having one huge file with everything in it, where it would be difficult to find out what what any one piece does.

If we use functions we can give them a name that reminds us of what the code does. For example, `setupGrid()` in the previous projects, did exactly that, it created and initialised a Pixel Grid. This makes it easier to see the bigger picture of what the entire program does.

Tip

Try to pick function names that tell you what the function does.

Sometimes you will be looking at your code again (many months later) and this helps!

All functions have the following format.

function addNumbers(number1, number2) {
  // Any amount of statements between { and }
  // Optionally a return with a value
  return  number1 + number2; 
}

Returning

A return statement allows us to get a result of the function, after the function finishes.

var a = 20;
var b = 23;

var result = addNumbers(a,b);
console.log("Result is " + result);

What is the output of the following code?

function one() {
  var a = 1;
  console.log( a );
}

function two() {
  var a = 2;
  console.log( a );
}

one();
two();
